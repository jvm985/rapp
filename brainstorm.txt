r app
------
bouw iets soortgelijks als rstudio, plaats het in /home/joachim/develop/R
maar veel eenvoudiger
het scherm bestaat uit: code (links), feedback van r (eronder), variabelen, files, ... rechtsboven, en grafische output rechtsonder
user logt in met google, zoek in /home/develop/opdrachten hoe dat moet, doe exact hetzelfde
alles moet in een docker komen, deployed op r.irishof.cloud (met letsencrypt)
maak de nodige nginx config
code pushen op github, en deployen op productie
user moet bestanden kunnen bewaren en uploaden, doe dit met mongo of zoiets
user moet bestanden kunnen delen met andere users (keuze r/w)
zorg dat users tegelijk in een bestand kunnen werken, zoals google docs
sommige users zijn beheerder, die moeten user beheer kunnen doen, en alle bestanden zien, en andere beheerders toevoegen
joachim.vanmeirvenne@atheneumkapellen.be is beheerder
maak een test@gemini.com user/beheerder  zodat jij bij alles inloggen en testen

- indeling scherm: twee kolommen, twee rijen. linker kolom: boven r code (met tabs die aangeven in welk bestand je werkt), onder r output. rechter kolom : boven een file manager, onderaan visualisatie en variabelen (zoals het nu is)
- de groottes van de vensters moeten aanpasbaar zijn
- de output van R niet centreren
- de filemanager moet ook met mappen om kunnen: dus als ik er op klik, moet je de inhoud van die map tonen. voorzie bovenaan een navigatie
- dit werkt niet goed: welke r code uitvoeren: als code is geselecteerd, enkel de selectie. als niets is geselecteerd, enkel de lijn waar de cursor staat.
- installeer alle packages (toegankelijk voor iedereen) die je ziet op https://support.posit.co/hc/en-us/articles/201057987-Quick-list-of-useful-R-packages
- voorzie een knop om output te wissen
- de output van r is niet mooi: er komt altijd iets bij als null device en een 1: fix dat
- maak deploen proper: deploy alles naar github, dan naar productie en build daar
- filemanager: voorzie per file: knippen, plakken, kopieren, delen, naam wijzigen, downloaden
- filemanager: voorzie een tablad voor bestanden die zijn gedeeld met mij door andere gebruikers. deze bestanden mag ik alleen kunnen openen en bewerken, dus niet delen, naam wijzigen, verwijderen, kopieren
- filemanager: uploaden, nieuwe map, nieuw bestand
- plots: voorzie download
- plots: voorzie navigatie naar vorige-volgende plot, en ook een wis alles knop
- voorzie een favicon R
- titel van de app is R, niet R editor. voorzie dit ook voor titelvenster browser
- schrijf een volledige technische documentatie van de R app, zodat je volgende sessie dat kan lezen en direct mee bent met alles. bewaar dat in technisch.txt
- R editor: toon de naam van het bestand (tab) in het rood wanneer niet bewaarde wijzigingen. Maar bewaar altijd alles, zodat er nooit wijzigingen verloren gaan. Dus: een gebruiker kan een bestand hebben dat niet bewaard is, maar wanneer hij uitlogt en weer in, moet datzelfde niet bewaarde bestand er zijn. Het punt is dat wanneer een andere gebruiker dat bestand opent, hij de wijzigingen niet kan zien van de eigenaar. Snap je?
- R console: de titel en wisknop blijven altijd zichtbaar, zodat je er niet voor naar boven moet scrollen
- plots: als R een nieuwe plot genereert, spring in het venster dan naar die nieuwe plot
- filemanager: uploaden knop voor bestanden van lokale schijf up te loaden; meerdere bestanden en mappen uploaden mogelijk maken
- filemanager: ipv voor elk bestanden iconen, doe het zoals in het screenshot bestand dat ik hier plaatste
- filemanager: daar moet je de bestanden niet in het rood tonen wanneer ze zijn gewijzigd
- wanneer ik library() gebruik in R, moet die in de sessie environment blijven
- in code editor: als ik op ctrl-enter druk, moet je de uidige lijn naar R sturen.
