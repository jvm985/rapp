r app
------
bouw iets soortgelijks als rstudio, plaats het in /home/joachim/develop/R
maar veel eenvoudiger
het scherm bestaat uit: code (links), feedback van r (eronder), variabelen, files, ... rechtsboven, en grafische output rechtsonder
user logt in met google, zoek in /home/develop/opdrachten hoe dat moet, doe exact hetzelfde
alles moet in een docker komen, deployed op r.irishof.cloud (met letsencrypt)
maak de nodige nginx config
code pushen op github, en deployen op productie
user moet bestanden kunnen bewaren en uploaden, doe dit met mongo of zoiets
user moet bestanden kunnen delen met andere users (keuze r/w)
zorg dat users tegelijk in een bestand kunnen werken, zoals google docs
sommige users zijn beheerder, die moeten user beheer kunnen doen, en alle bestanden zien, en andere beheerders toevoegen
joachim.vanmeirvenne@atheneumkapellen.be is beheerder
maak een test@gemini.com user/beheerder  zodat jij bij alles inloggen en testen

verbeterpunten
--------------
- de groottes van de vensters moeten aanpasbaar zijn
- welke r code uitvoeren: als code is geselecteerd, enkel de selectie. als niets is geselecteerd, enkel de lijn waar de cursor staat.
- indeling scherm: twee kolommen, twee rijen. linker kolom: boven r code (met tabs die aangeven in welk bestand je werkt), onder r output. rechter kolom : boven een file manager, onderaan visualisatie en variabelen (zoals het nu is)
- de filemanager moet ook met mappen om kunnen, plus alles wat hij nu al kan
- de output van R niet centreren
